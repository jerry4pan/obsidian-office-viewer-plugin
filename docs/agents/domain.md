# Domain documentation

This is a single-context repository.

## Before exploring or changing the project

- Read `CONTEXT.md` at the repository root when it exists.
- Read ADRs under `docs/adr/` that affect the area being changed.
- If either location does not yet exist, proceed without treating its absence as an error.

## Vocabulary

Use domain terms as defined in `CONTEXT.md` in issue titles, requirements, tests, code, and documentation. If a required concept is missing, call out the gap rather than silently inventing competing terminology.

## Architecture decisions

If proposed work contradicts an existing ADR, identify the conflict explicitly and explain why the decision may need to be reopened. Do not silently override an ADR.
